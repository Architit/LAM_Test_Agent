## 1. OBJECTIVE
To enforce a sequential, contract-bound execution of all development phases (A, B, C, D, E, F) across all 24 Sovereign Trees and supporting ecosystem repositories.

## 2. EXECUTION RULES
1. **Sequentiality:** Repositories are processed one-by-one according to the Wave priority.
2. **Phase Gating:** No repository can move to the next phase until the current phase's "Owner Closure Evidence" is signed and verified.
3. **Completeness:** A repository is considered "AWAKENED" only after Phase F completion.
4. **Consistency:** All repositories must adopt the canonical ARGD triad (True Name, Call Sign, System ID).

## 4. PHASE DoD (Definition of Done)
- **Phase A (Stabilization):** Repository structure normalized, `IDENTITY.md` present, basic git hygiene.
- **Phase B (Quality):** Smoke tests replaced by contract tests, `ROADMAP.md` and `DEV_LOGS.md` active.
- **Phase C (Memory):** Memory-mode wiring and state persistence contracts.
- **Phase D (Transport):** Inter-repo communication and routing contracts.
